# AIRP v0.7 Extended Optimization Mandate

This file outlines the continued execution of the v0.7 optimization and efficiency phase. The primary goal remains to aggressively reduce technical debt, eliminate "God Classes" and "Mega Widgets", and establish clean, scalable service boundaries.

## Core Mandates
1.  **Iterative Stability:** Preserve application stability. All changes must be incremental, fully tested, and maintain existing functionality.
2.  **Strict Versioning:** Every discrete architectural slice must be committed with a version bump of `0.0.1` (e.g., 0.7.8, 0.7.9).
3.  **Analyzer Compliance:** Zero static analysis errors or warnings before any commit.
4.  **Test Integrity:** The test suite must pass 100% after every slice.

## Extended Priority Backlog (Ordered)

### Slice 0.7.8: Visuals & Atmosphere Restoration
**Objective:** Restore the background image gallery and atmospheric controls accidentally removed in previous refactors.
- [x] Restore background selection gallery to `VisualSettingsPanel`.
- [x] Restore "CLEAR BACKGROUND" / "RESET TO DEFAULT" logic for background assets.
- [x] Restore "Loading Animation" and "Bloom" toggles to `VisualSettingsPanel`.
- [x] Re-integrate `image_picker` and `FileIOHelper` image rendering in the settings panel.

### Slice 0.7.9: AI Provider Strategy Pattern
**Objective:** Eliminate massive `switch` statements and tight coupling to specific AI providers in `ModelRegistryService` and `ChatApiService`.
- [x] Define a core `AiProviderStrategy` interface.
- [x] Implement concrete strategies for major ecosystems (e.g., `GeminiStrategy`, `OpenAiCompatibleStrategy`, `OpenRouterStrategy`, `AnthropicStrategy`).
- [x] Refactor `ModelRegistryService` to delegate URL generation, header formatting, and JSON parsing to these strategies.
- [x] Refactor `ChatApiService` (specifically `fetchModels` and `streamResponse` methods) to utilize the strategies instead of hardcoded conditional logic.

### Slice 0.7.10: Streaming Coordinator Service Extraction
**Objective:** Remove background stream management and message chunk parsing from `ChatProvider`.
-   Create a `StreamingCoordinatorService`.
-   Migrate `_activeStreams`, `_activeNotifiers`, `_activeStreamTexts`, and `_activeStreamModels` from `ChatProvider` to this new service.
-   Migrate the complex chunk parsing, error handling, and `BackgroundNotification` logic out of `ChatProvider`.
-   `ChatProvider` should only initiate streams via the coordinator and listen for state changes.

### Slice 0.7.11: Settings Provider Extraction
**Objective:** Isolate application-wide settings (temperature, topP, grounding, search provider config) from the chat state.
-   Create a `SettingsProvider` (or `AppConfigProvider`).
-   Move properties like `_temperature`, `_topP`, `_topK`, `_maxOutputTokens`, `_enableGrounding`, `_searchProvider`, `_searxngUrl`, etc., out of `ChatProvider`.
-   Update UI components to consume these settings from the new provider instead of `ChatProvider`.

### Slice 0.7.12: Settings Drawer Decomposition & Reactivity
**Objective:** Break down the `SettingsDrawer` "Mega Widget" and replace manual controller synchronization with reactive state.
-   Decompose `SettingsDrawer` into focused sub-panels (e.g., `ApiKeysPanel`, `ModelConfigPanel`, `AppearancePanel`, `BehaviorPanel`).
-   Refactor the UI to use reactive bindings (e.g., `Selector`, `Consumer`, or direct `ValueNotifier` listeners) bound directly to the `SettingsProvider` and `ApiKeyService`, eliminating the need for dozens of local `TextEditingController` instances and the fragile `_syncControllers` / `_handleSaveSettings` methods.

### Slice 0.7.13: Theme & VFX Provider Split
**Objective:** Separate pure UI styling from performance-intensive visual effects state.
-   Extract VFX logic (animated motes, custom background image loading, bloom toggle) from `ThemeProvider`.
-   Create a dedicated `VfxProvider` or `BackgroundProvider` to manage these heavy rendering configurations.
-   Ensure `ThemeProvider` focuses solely on colors, typography, and standard Material properties.
